# 2026-03-12

## OpenClaw tool execution fix (root cause)

**Problem:** OpenClaw returned tool-call text (e.g. `<exec>echo X</exec>`) instead of actually executing tools. Spent a day debugging.

**Root cause:** `tools.allow: ["group:web"]` in `~/.openclaw/openclaw.json` *restricts* tools to only web_search and web_fetch. Exec, read, write, and all other coding tools were filtered out. The model received zero tools, so it output text instead of structured tool calls.

**Fix:** Changed `tools.allow` to `tools.alsoAllow: ["group:web"]`. This *adds* web tools to the coding profile without removing exec, read, write, etc.

**Verification:** Logs now show `toolsRaw=15 finalTools=15` including exec, read, write, process. OpenClaw can execute tools.

**Docs updated:** OPENCLAW_INTEGRATION.md, OPENCLAW_DIAGNOSIS.md.
